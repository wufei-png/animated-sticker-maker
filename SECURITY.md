# Security Policy

## Supported version

Security fixes are applied to the latest release on the `main` branch.

## Reporting a vulnerability

Please use GitHub's private security advisory flow for this repository. Do not include sensitive paths, private reference images, access tokens, or unpublished sticker assets in a public issue.

When reporting, include the affected script and version, a minimal reproduction using non-sensitive assets, the expected impact, and any suggested mitigation. Reports concerning path traversal, unsafe file replacement, malformed image handling, or artifact-integrity bypasses are especially useful.
