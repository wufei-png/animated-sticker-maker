# Use one read-only doctor entry point

Expose `scripts/doctor.py` as the single validation and diagnosis entry point. With no subcommand it diagnoses `.` as one unambiguous artifact boundary; explicit `motion`, `package`, `report`, and `export` subcommands select a narrower target. The command never recursively scans arbitrary directories and never mutates or repairs artifacts. It re-runs every deterministic schema, media, path, state, and binding check available from the artifact, aggregates findings, and reports `healthy`, `incomplete`, or `invalid`. Human-readable output is the default, while a versioned JSON envelope with stable check IDs serves agents and CI. This keeps one operator-facing mental model while preserving explicit, deterministic boundaries for automation.
